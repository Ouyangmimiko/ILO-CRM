2. System Overview

The ILO CRM is a system developed specifically for the Industry Liaison Office (ILO) team, mainly used for managing and tracking industry partnerships. The user interface of the system is divided into two main modules: User Management and Engagement Management, allowing users to perform different management tasks.

3. User Roles and Permissions

Roles:

Master Admin:

  • Has the highest level of access. The Master Admin's sidebar shows both the Engagement Management and User Management modules.
  • The Master Admin can perform all management operations, including system user management, role management, database management, and project management. Only this role can perform permission-related operations (e.g., viewing and switching roles).

Normal Admin:

  • The Normal Admin's sidebar only displays the Engagement Management module, without the User Management.
  • On the Engagement Management page, the Normal Admin can perform CRUD operations, including adding, deleting, viewing, and editing contact information.
  • The Normal Admin cannot access the User Management module and therefore cannot manage users or roles.

User:

  • The User's sidebar also only displays the Engagement Management module.
  • On the Engagement Management page, the User can only browse contact information, without any add, delete, or edit permissions (i.e., no corresponding buttons are available).
  • The User cannot access any management functions, including user and role management.

4. Settings

Accessible Users: All three user types (Master Admin, Normal Admin, User) have access to this module.

Functionality: The Settings module allows users to configure some basic system settings, such as the range of the year selector. This module is designed to let all users adjust specific parameters as needed to ensure that the system meets current requirements.

Steps:

Accessing the Settings Page:

  • After logging into the system, click the gear icon in the upper right corner ("App Setting") to enter the settings page.
  • The page displays the "Config the year selector" option, allowing users to adjust the default end year range and the default start year range of the year selector.

Adjusting the Year Range:

  • Default End Year Range: Use the plus or minus buttons to adjust the default end year range of the year selector. This setting determines the end year selected in other modules.
  • Default Last Year Range: Use the plus or minus buttons to adjust the default start year range of the year selector. This setting determines the start year selected in other modules.

Saving Settings:

  • After adjustments, the system will automatically save the settings, and all operations related to the year selector will take effect based on the new settings.

5. User Center

Accessible Users: All three user types (Master Admin, Normal Admin, User) have access to this module.

Functionality: The User Center module allows users to view and manage their personal information, including username, email address, and password. All users can access this module to update their personal profile information or change their password.

Steps:

Accessing the User Center Page:

  • After logging into the system, click the user icon or username in the upper right corner, and select "User Center" to enter the user center page.

Viewing User Information:

  • On the User Center page, users can view their current username, email address, and user role (such as Master Admin, Normal Admin, or User).

Editing User Information:

  • Click the "Edit" button to edit the username. After editing, click the save button to confirm the changes.

Changing Password:

  • Click the "Change Password" button to enter the password change page.
  • Enter the current password, then enter the new password and confirm it. Click the confirm button to complete the password change.

Logging Out:

  • In the upper right corner of the user menu, click "Log out" to exit the system.

6. Engagement Management

Accessible Users: Master Admin and Normal Admin can fully operate, User can only browse.

Functionality: The Engagement Management module is mainly used to manage and track interactions and contact information with industry partners. This module allows users to view, add, edit, and delete contact information, as well as import and export data.

Viewing Contact Information:

  • All users (including Master Admin, Normal Admin, and User) can view detailed contact information.
  • On the "Master Database" page, the system displays a table containing all contacts. Click the arrow icon at the front of each row to expand and view detailed information about the contact.
  • After expanding, users can see the contact's organization, name, job title, email address, alumni status, and participation in research projects.
  • Other Engagement Details: After checking this option, users can view the contact's other engagement details, such as interaction records with other projects.
  • Show Status by Selected Years: After checking this option, users can view the contact's participation status within the selected year range. This includes mentoring status, industry involvement, and client project status during specific years.

Searching Contacts:

  • At the top of the page, users can search in two ways:
  • All: If "All" is selected, users can directly enter keywords in the subsequent input box and click the "Search" button. The system will search for matching items based on the keywords across all contact information.
  • Specific: If "Specific" is selected, clicking the "Search" button will open a search window. In the pop-up window, users can enter search criteria in the corresponding fields, including organization, first name, surname, job title, email address, alumni status, and research projects. Users can also check options below to specify more detailed search criteria, such as mentoring status, industry participation status, client project status, and other engagement details. After entering the information, click the "Search" button in the pop-up window to perform the search.

Adding/Editing Contacts:

  • Master Admin and Normal Admin can add and edit contacts.
  • When adding or editing a contact, clicking the "Add" or "Edit" button will bring up a form page.
  • On the form page, users need to enter basic contact information, such as organization name, first name, surname, and email address. Fields marked with a red asterisk (*) are mandatory.
  • Below the form, there are three optional status boxes: Mentoring Status, Year in Industry Status, and Project Client Status:
  • Mentoring Status: Checking this box will activate and allow editing of the contact's mentoring participation across different years (e.g., 2023-2024, 2024-2025).
  • Year in Industry Status: Checking this box will activate and allow editing of the contact's industry participation across different years.
  • Project Client Status: Checking this box will activate and allow editing of the contact's project client participation across different years.
  • Users can check and edit these statuses as needed to record the contact's full participation details.
  • After entering all necessary information, click the "Confirm Add" or "Confirm Edit" button to save the contact information.
  • If the user unchecks a status box while editing, the corresponding editing fields will be disabled, and any previously entered data will not be saved.
  • Note: If a duplicate email address is entered while manually adding a new record, the system will display a message: "The email has already been taken." In this case, the user should:
  • Cancel the current add operation.
  • Use the Specific Search or All search option to enter the duplicate email address and click the "Search" button.
  • The system will display the contact record associated with that email.
  • Locate the record and click the "Edit" button to enter edit mode and make necessary changes.

Uploading Contact Data:

  • Master Admin and Normal Admin can click the "Upload Excel" button, select an Excel file containing contact information, and the system will batch import these contact records.
  • User cannot see or use the "Upload Excel" button.

Exporting Data:

  • All users can use the "Export" button to export the currently filtered or queried contact data, generating an Excel file or other format report.

7. User Management

Accessible Users: Only Master Admin

Functionality: The User Management module is divided into User Management and Role Management sections. The Master Admin can use this module to manage all users in the system, including creating new users, assigning roles, updating user information, and deleting users.

User Management:

Viewing User List:

  • After entering the "User Management" page, the Master Admin can view a list of all users in the system. The list displays each user's username, email address, and user role (Admin or Not Admin).
  • The user list also includes operation options for managing existing users.

Creating New User:

  • By clicking the "Create New User" button, the Master Admin can enter basic information for a new user, including username, email address, and initial password.
  • Once created, the new user will be added to the user list and assigned a default role.

Deleting User:

  • In the user list, the Master Admin can delete existing users by clicking the "Delete" button.
  • Deletion requires confirmation, and once deleted, the user will be permanently removed and cannot be recovered.

Role Management:

Viewing and Switching Roles:

  • On the "Role Management" page, the Master Admin can view the role status of all users.
  • The role of each user can be switched between "Normal user" and "Administrator." Once the role is switched, it will take effect immediately, updating the user's access permissions.

Assigning Roles:

  • The Master Admin can assign appropriate roles to new users. By default, users may be assigned the Normal user role, and the admin can adjust it to Administrator role as needed.
